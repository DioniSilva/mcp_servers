# Shared Catalog MCP Server

Local MCP server for sharing `skills`, `tools` definitions, `prompts`, and `resources` across coding tools. The v1 server is a read-only catalog: it does not execute commands, scripts, or local tools.

## Requirements

- Node.js 20+
- npm

## Development

```bash
npm install
npm run build
npm test
```

During development:

```bash
npm run dev
```

After building:

```bash
npm start
```

## Installation via GitHub Release

The recommended way to distribute this server without publishing to the npm registry is to attach the tarball generated by `npm pack` to a GitHub Release.

Install or update the latest `shared-catalog` release with the repository script:

```bash
scripts/install-latest-shared-catalog.sh
```

Install or update without cloning the repository:

```bash
curl -fsSL https://raw.githubusercontent.com/DioniSilva/mcp_servers/main/scripts/install-latest-shared-catalog.sh | bash
```

Install from a `.tgz` file downloaded from the release:

```bash
npm install -g ./shared-catalog-mcp-server-0.1.0.tgz
shared-catalog-mcp
```

Install directly from the release asset URL:

```bash
npm install -g https://github.com/DioniSilva/mcp_servers/releases/download/shared-catalog-v0.1.0/shared-catalog-mcp-server-0.1.0.tgz
```

After global installation, configure MCP clients to use the binary:

```json
{
  "mcpServers": {
    "shared-catalog": {
      "command": "shared-catalog-mcp",
      "args": []
    }
  }
}
```

## Release

Releases are automated by GitHub Actions on pushes to the repository `main` branch. The workflow validates the package, builds the npm tarball, and creates a GitHub Release if `shared-catalog-vX.Y.Z` does not already exist.

Before changing release behavior, run:

```bash
npm run release:check
```

To generate the package locally:

```bash
npm run release:pack
```

The command creates a file such as `shared-catalog-mcp-server-0.1.0.tgz`. This file is a local artifact and should not be committed.

Release checklist:

1. Update `version` in `package.json`.
2. Run `npm install` to synchronize `package-lock.json` when the version changes.
3. Run `npm run release:check`.
4. Push or merge the change to `main`.
5. Confirm the GitHub Actions workflow creates `shared-catalog-vX.Y.Z`.

## MCP Capabilities

The server exposes five safe tools:

- `catalog.list`: lists items by `kind`, `tag`, and optional text query.
- `catalog.get`: returns a complete catalog item by `id`.
- `catalog.search`: searches item names, descriptions, tags, and content.
- `catalog.render`: returns content ready to use with normalized metadata.
- `catalog.update_info`: returns read-only install and update information for this MCP server.

It also exposes resources with the `catalog://{kind}/{id}` URI format and versioned catalog prompts as native MCP prompts.

## Catalog Structure

```text
catalog/
  skills/*.md
  tools/*.yaml
  prompts/*.md
  resources/**/*
```

Markdown files use YAML frontmatter. Example:

```md
---
id: code-review-checklist
name: Code Review Checklist
description: Reusable checklist for reviewing code changes.
tags: [review, quality]
version: 0.1.0
kind: skill
contentPath: catalog/skills/code-review-checklist.md
---

Skill content.
```

Tool files are full YAML documents that describe a reusable interface without execution:

```yaml
id: repo-search
name: Repository Search
description: Text search in repository files.
tags: [repo, search]
version: 0.1.0
kind: tool
contentPath: catalog/tools/repo-search.yaml
inputSchema:
  type: object
  properties:
    query:
      type: string
  required: [query]
clientHints:
  execution: "not-supported"
```

## Client Configuration

If you are using the global installation from a release, prefer `shared-catalog-mcp` as the command. For local development, replace `/ABS/PATH/mcp_servers` with the absolute path to this workspace root.

### Codex

With global installation:

```toml
[mcp_servers.shared_catalog]
command = "shared-catalog-mcp"
args = []
```

During local development, point to the build output:

```toml
[mcp_servers.shared_catalog]
command = "node"
args = ["/ABS/PATH/mcp_servers/servers/shared-catalog/dist/index.js"]
```

### GitHub Copilot / Copilot Chat

When the environment supports MCP over `stdio`, use an equivalent configuration:

```json
{
  "mcpServers": {
    "shared-catalog": {
      "command": "node",
      "args": ["/ABS/PATH/mcp_servers/servers/shared-catalog/dist/index.js"]
    }
  }
}
```

### OpenCode

```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "shared-catalog": {
      "type": "local",
      "command": ["node", "/ABS/PATH/mcp_servers/servers/shared-catalog/dist/index.js"],
      "enabled": true
    }
  }
}
```

## Adding Items

1. Create the file in the correct `catalog/` subdirectory.
2. Fill in the required metadata: `id`, `name`, `description`, `tags`, `version`, `kind`, `contentPath`.
3. Run `npm test` to validate schemas and indexing.
4. Run `npm run build` before using the server in MCP clients.

## Manual Test via stdio

After `npm run build`, configure an MCP client with:

```bash
node /ABS/PATH/mcp_servers/servers/shared-catalog/dist/index.js
```

In the client, call `catalog.list`, `catalog.search`, `catalog.get`, or read a URI such as `catalog://skill/code-review-checklist`.
