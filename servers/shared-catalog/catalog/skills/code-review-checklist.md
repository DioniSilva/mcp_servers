# Code Review Checklist

Use this skill to review changes with a focus on behavior, risks, and tests.

- Start with bugs, regressions, and contract risks.
- Check whether tests cover the changed behavior.
- Prefer objective comments with file and line when there is a clear action.
- Keep summaries and praise out of the way of the main findings.
